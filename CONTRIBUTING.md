# Contributing

Thank you for your interest in contributing to Cybersecurity Principles.

## Guidelines

- Keep content vendor-neutral whenever possible.
- Focus on security principles rather than specific tools.
- Use clear and concise language.
- Cite references when appropriate.
- Do not copy copyrighted material from books, standards, benchmarks, or proprietary documentation.
- Ensure examples are original or publicly available.

## Documentation Style

- Use Markdown.
- Use descriptive headings.
- Keep sections focused on a single concept.
- Include practical examples whenever possible.
- Include security review questions where applicable.

## Pull Requests

Before submitting a pull request:

- Verify Markdown formatting.
- Check for broken links.
- Ensure content aligns with the repository's objectives.
- Ensure workflows pass successfully.

Thank you for helping improve this project.
