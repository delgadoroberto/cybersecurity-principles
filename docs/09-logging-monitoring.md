# Logging and Monitoring

## Overview

Logging and Monitoring are fundamental cybersecurity capabilities that
enable organizations to detect, investigate, and respond to security
events.

Without visibility into systems, networks, applications, and cloud
environments, organizations cannot effectively identify malicious
activity, operational failures, or security incidents.

Logging provides the data.

Monitoring provides the visibility.

Together, they form the foundation of security detection and response.

---

## Why It Matters

Security incidents rarely occur without leaving evidence.

Attackers often generate indicators that can be observed through logs,
alerts, and monitoring systems.

Examples include:

- Failed authentication attempts
- Privilege escalation activities
- Unauthorized access attempts
- Configuration changes
- Malware execution
- Data exfiltration attempts
- Network anomalies

Organizations that lack adequate logging and monitoring may discover
security incidents long after the damage has occurred.

Effective monitoring reduces detection time and supports faster response.

---

## Core Concepts

### Logging

Logging is the process of recording events generated by systems,
applications, devices, and services.

Examples include:

- User logins
- System events
- Application events
- Network activity
- Administrative actions
- Security events

Logs provide historical evidence of activity.

### Monitoring

Monitoring is the continuous observation and analysis of collected data.

Monitoring helps organizations:

- Detect suspicious behavior
- Identify operational issues
- Generate alerts
- Support investigations
- Improve visibility

### Alerting

Alerting occurs when monitoring systems identify conditions that require
attention.

Examples:

- Multiple failed login attempts
- Privileged account activity
- Unauthorized configuration changes
- Malware detection events

Not every event should generate an alert.

Effective alerting reduces noise and improves response efficiency.

---

## Types of Logs

Organizations typically collect logs from multiple sources.

### System Logs

Generated by operating systems.

Examples:

- User authentication
- Service events
- System startup and shutdown
- Administrative actions

### Application Logs

Generated by applications and services.

Examples:

- User activity
- API requests
- Errors and exceptions
- Administrative actions

### Network Logs

Generated by network infrastructure.

Examples:

- Firewall activity
- VPN connections
- DNS queries
- Proxy activity

### Security Logs

Generated by security technologies.

Examples:

- Endpoint protection tools
- Identity platforms
- Vulnerability scanners
- Intrusion detection systems

### Cloud Logs

Generated by cloud platforms and services.

Examples:

- Resource creation
- Access events
- Configuration changes
- Administrative actions

---

## Security Monitoring Objectives

Security monitoring should support several objectives.

### Detect Malicious Activity

Identify indicators of compromise and suspicious behavior.

### Detect Policy Violations

Identify actions that violate organizational security requirements.

### Detect Unauthorized Access

Monitor authentication and authorization events.

### Detect Configuration Changes

Identify unexpected modifications to systems and services.

### Support Incident Response

Provide visibility during investigations.

### Support Compliance Requirements

Many frameworks require logging and monitoring capabilities.

---

## Characteristics of Effective Logging

Not all logging provides useful security visibility.

Effective logging should be:

### Accurate

Events should be recorded correctly.

### Complete

Relevant events should not be omitted.

### Consistent

Logging formats should be standardized when possible.

### Timely

Events should be available quickly for analysis.

### Protected

Logs should be protected against unauthorized modification.

### Retained

Logs should be stored according to business and regulatory requirements.

---

## Centralized Logging

Organizations often aggregate logs into centralized platforms.

Benefits include:

- Improved visibility
- Simplified investigations
- Correlation across systems
- Reduced operational complexity
- Better reporting capabilities

Centralized logging helps security teams analyze events from multiple
sources in a single location.

---

## Security Information and Event Management (SIEM)

Many organizations use SIEM platforms to support monitoring.

A SIEM typically provides:

- Log collection
- Event correlation
- Alerting
- Reporting
- Investigation support

SIEM platforms help security teams identify patterns that may not be
visible when analyzing individual logs.

## Detection Use Cases

Monitoring programs should focus on meaningful detection scenarios.

Examples include:

### Authentication Attacks

- Brute-force attacks
- Credential stuffing
- Password spraying

### Privilege Escalation

- Administrative account misuse
- Unauthorized privilege changes

### Lateral Movement

- Unexpected remote access
- Internal reconnaissance

### Data Access

- Sensitive data access
- Unusual data transfers
- Data exfiltration attempts

### Cloud Activity

- Resource creation
- Permission changes
- Unusual administrative actions

---

## Common Monitoring Challenges

### Excessive Logging

Collecting everything can create excessive costs and operational burden.

### Insufficient Logging

Missing critical events reduces visibility.

### Alert Fatigue

Too many alerts can overwhelm security teams.

### Poor Log Quality

Incomplete or inaccurate logs reduce effectiveness.

### Lack of Context

Events without business context may be difficult to interpret.

### Limited Retention

Short retention periods may impact investigations.

---

## Relationship to Other Security Principles

Logging and Monitoring support multiple cybersecurity principles.

### CIA Triad

Monitoring helps protect confidentiality, integrity, and availability.

### Defense in Depth

Monitoring provides an additional security layer.

### Least Privilege

Access events help validate privilege usage.

### Vulnerability Management

Monitoring may identify exploitation attempts.

### Incident Response

Logs provide critical evidence during investigations.

## Real-World Example

Consider a cloud-hosted business application.

### Event

An attacker attempts to access an administrative account.

### Logged Activity

- Failed login attempts
- Geographic anomalies
- Privileged access requests

### Monitoring Response

The monitoring platform identifies unusual behavior and generates an
alert.

### Outcome

Security analysts investigate the activity before compromise occurs.

Early detection reduces organizational risk.

---

## Common Mistakes

### Logging Without Monitoring

Collecting logs alone provides limited value.

### Monitoring Without Defined Use Cases

Monitoring should align with specific detection objectives.

### Ignoring Log Integrity

Attackers may attempt to modify or delete logs.

### Retaining Logs for Too Short a Period

Historical data is often required for investigations.

### Generating Too Many Alerts

Excessive alerting reduces effectiveness.

### Failing to Review Alerts

Unreviewed alerts provide no security benefit.

## Benefits

Effective Logging and Monitoring provide:

- Improved visibility
- Faster threat detection
- Better incident response
- Enhanced investigations
- Improved compliance
- Reduced attacker dwell time
- Better operational awareness

---

## Key Takeaways

- Logging records events.
- Monitoring analyzes events.
- Both capabilities are essential for cybersecurity operations.
- Visibility is a prerequisite for detection and response.
- Alert quality is more important than alert quantity.
- Centralized logging improves security operations.
- Effective monitoring supports resilience and risk reduction.

---

## Security Review Checklist

When evaluating a system, consider the following questions:

### Logging

- Are security events logged?
- Are authentication events recorded?
- Are administrative actions logged?
- Are logs protected?

### Monitoring

- Are logs actively monitored?
- Are alerts configured?
- Are monitoring use cases defined?

### Alerting

- Are alerts actionable?
- Are alerts reviewed?
- Is alert fatigue managed?

### Retention

- Are logs retained appropriately?
- Are retention requirements documented?
- Can historical data support investigations?

### Visibility

- Is monitoring centralized?
- Are critical systems covered?
- Are cloud services included?
